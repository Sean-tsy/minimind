## 1. 预训练评估的不稳定性

### 1.1 核心问题

任何基于 checkpoint 的诊断框架面临的基础性挑战是：**预训练过程中的评估本身就是不稳定的**。MaP 框架（Wang et al., 2025, arXiv:2510.09295）对这一问题提供了最系统的诊断。

### 1.2 MaP 框架：双源不稳定性分解

MaP 将整体评估不稳定性分解为两个独立来源：

1. **参数不稳定性（Parameter Instability）**：来自训练随机性。LLM 在高维参数空间中的训练轨迹天然不平滑，任何单一 checkpoint 都可能代表一个暂时次优状态或尖锐局部极小值
2. **评估不稳定性（Evaluation Instability）**：来自脆弱的测量协议。对于代码生成、数学推理等生成式任务，基于单次输出的指标（如 greedy decoding）本质上是高方差的伯努利试验

**解决方案**：
- **Checkpoint Merging**：平均近期多个 checkpoint 的权重，平滑参数空间中的高频波动
- **Pass@k Metric**：用多次采样的统计估计取代单次输出的评估

**实验结果**：在 1B、4B、8B 模型上，组合框架将 Kendall 秩相关系数从 0.635 提升至 0.799（1B dense 模型平均），显著提升了性能轨迹的平滑度和模型排名的一致性。

### 1.3 对本框架的启示

| MaP 发现 | 本框架的对应设计 |
|---|---|
| 单一 checkpoint 评估不可靠 | Module 4 跨多个阶段转换比较，而非孤立评估 |
| 参数不稳定性导致性能波动 | Module 4 参数漂移分析直接测量参数层面变化 |
| 评估协议影响结论 | 统一使用 Gemini 评分 + 结构化 prompt 集减少测量噪声 |
| Checkpoint merging 可作为去噪工具 | 可选：在诊断前对相邻 checkpoint 做 weight averaging |

**关键结论**：Loss 曲线是一维信号，MaP 形式化了本框架背后的核心直觉——可靠评估需要同时平滑模型参数和测量协议。

## 2. 灾难性遗忘：机制与度量

灾难性遗忘是 Module 2（能力保留测试）的核心检测目标。文献从机制分析、实证研究、概念级动态三个层面提供了全面的理论基础。

### 2.1 机制分析：梯度干扰、表征漂移、损失景观平坦化

**Laitinen & Imanov (2026, arXiv:2601.18699)** 对 109B–1.5T 参数规模的 Transformer LLM 进行了最全面的遗忘机制分析，识别出三个主要遗忘机制：

| 遗忘机制 | 具体表现 | 量化指标 |
|---|---|---|
| 梯度干扰 | 低层 15–23% 的 attention heads 出现严重扰动 | 与早期遗忘高度相关 |
| 表征漂移 | 中间层 CKA 相似度下降 0.32–0.47 | 表征空间重组 |
| 损失景观平坦化 | 先前任务极小值附近曲率下降 | 参数更新后景观变平 |

**核心发现**：任务间梯度对齐度可以预测遗忘严重程度（r = 0.87）。

**对 Module 4 的直接启示**：
- Fig 13（参数漂移热力图）操作化了他们识别的 attention 层扰动信号
- Fig 14（表征相似度曲线）操作化了中间层 CKA 下降的测量
- 按 Attn Q/K/V vs. FFN 分组的参数漂移分析，直接来源于该论文对功能模块级遗忘定位的方法

### 2.2 实证研究：持续指令微调中的遗忘

**Luo et al. (2023, arXiv:2308.08747)** 在 BLOOMZ 和 mT0 模型（1B–7B）上实证评估了持续指令微调中的遗忘：

**关键发现**：
- 在 1B–7B 范围内，**模型规模越大，遗忘越严重**——可能因为大模型初始性能更强，因此退化空间更大
- **通用指令微调在特定任务微调之前进行，有助于缓解遗忘**
- 遗忘度量定义：FG = (1/|E|) × Σ(R_initial - R_current) / R_initial × 100%

**对本框架的启示**：
- 即使在 MiniMind 的小模型规模下，遗忘也是可观测和可度量的
- 他们的归一化遗忘度量与我们 Module 2 的 forgetting_rate 公式高度一致
- 建议采纳其标准化公式以提高跨研究可比性

### 2.3 概念级学习动态

**Yao et al. (2025, arXiv:2601.03570)** 从全新视角研究了 LLM 如何在持续预训练中获取和遗忘概念：

- 提出 **Concept Circuits**——与特定概念关联的计算子图，并用图度量（节点重要性、冗余度、信息流效率、鲁棒性）表征其结构
- **结构权衡**：密集且鲁棒的电路结构有利于学习，但模块化结构更有利于抵抗遗忘
- **学习-遗忘正相关**：学习增益越大的概念，在后续训练中遗忘越严重
- **语义相似性驱动干扰**：语义相似的概念之间干扰更强

**Zucchet et al. (2025, arXiv:2503.21676)** 揭示知识获取的三阶段模式：

```
Phase 1: 初始语言理解       → 学习整体分布统计
Phase 2: 性能平台期         → 形成基于 attention 的检索电路
Phase 3: 知识涌现           → 学习个体特定的事实关联
```

**核心发现**：幻觉与知识同时涌现。将新知识通过微调整合进模型会迅速破坏其已有的参数记忆。

### 2.4 记忆感知的重放调度

**MSSR 框架 (arXiv:2603.09892)** 将认知记忆理论引入 LLM 持续学习：

- 基于时间衰减的保留函数调度重放，而非固定间隔
- 随着模型稳定性增加，逐步扩展重放间隔
- 在 GSM8K、MATH、MMLU 上展示了优越的保留-效率权衡

## 3. 对齐税与安全-能力权衡

"对齐税"（Alignment Tax）——安全对齐导致通用能力下降——是 Module 1 和 Module 2 评估 GRPO/DPO 阶段时的核心关切。

### 3.1 OGPSA：正交梯度投影

**Sun, Zhang et al. (2026, arXiv:2602.07892)** 将安全对齐重新定义为持续学习问题：

**核心论点**：对齐税本质上是异构持续学习中的灾难性遗忘——安全梯度无意中覆写了编码通用能力的参数子空间。

**OGPSA 方法**：
1. 用小规模通用数据集估计一个低秩的能力子空间 S_gen
2. 在安全对齐训练中，将安全梯度投影到 S_gen 的正交补空间
3. 产生的更新方向最小化对先验知识的干扰

**实验量化**（Qwen2.5-7B-Instruct, SFT→DPO）：

| 指标 | 标准对齐 | OGPSA |
|---|---|---|
| SimpleQA（事实性） | 0.53% | 3.03% |
| IFEval（指令遵循） | 51.94% | 63.96% |
| 安全拒绝率 | 高 | 保持高 |

**对本框架的启示**：
- Module 1 的 alignment_precision（安全拒绝率 - 误拒率）在行为层面测量对齐税
- Module 2 的保留矩阵在能力层面追踪各维度的对齐税
- 建议新增：**逐维度对齐税量化**——分别追踪 DPO 前后的事实知识、指令遵循质量、流畅性变化

### 3.2 对齐税的几何理论

**arXiv:2603.00047** 提供了基于主角度（principal angles）的对齐税几何框架：

- **可测预测**：每个任务的对齐税率 τ_i = ⟨v*, c_i⟩² 可在对齐训练前通过表示空间测量计算
- **可缩放诊断**：如果 τ_i 随模型维度下降 → 偶然重叠（工程问题，可通过扩大模型解决）；如果 τ_i 保持不变 → 内在重叠（基本权衡，需修改目标函数）
- **安全-安全前沿**：多个安全目标之间的权衡遵循与安全-能力前沿相同的几何结构

### 3.3 CoSMo-RL：VLM 的联合安全与能力优化

**CoSMo-RL (arXiv:2510.04196)** 为多模态模型提供了统一的 RL 训练框架：

- **两阶段训练**：先建立通用能力，再联合优化安全、价值和通用维度
- **多目标奖励**：Visual-Focus + Helpful + Format + Task-Aware
- **关键观察**：存在"过度安全"现象——对良性查询发出不必要的拒绝

这直接映射到我们 Module 3 的 over-alignment 检测：borderline prompt 拒绝率 > 40% → 检测到过度对齐。

## 4. VLM 特有挑战：遗忘、接地与幻觉

### 4.1 VLM 持续学习的三个核心失败模式

**Liu et al. (2025, arXiv:2508.04227)** 提供了 VLM 持续学习的首个全面综述，识别出三个核心挑战：

```
失败模式 1: 跨模态特征漂移 (Cross-Modal Feature Drift)
├── 现象：视觉和文本特征空间去同步化
├── 后果：多模态检索准确率下降，即使单模态分类性能稳定
└── 对应诊断：Module 4 — 跨模态对齐度量 (Fig 15)

失败模式 2: 共享模块干扰 (Shared Module Interference)
├── 现象：共享 cross-attention 层中，新任务梯度破坏性覆写旧任务关键权重
├── 后果：融合型架构的瓶颈效应
└── 对应诊断：Module 4 — Backbone drift comparison (Fig 17)

失败模式 3: 零样本能力侵蚀 (Zero-Shot Capability Erosion)
├── 现象：持续微调扭曲预训练嵌入分布
├── 后果：即使轻微的领域偏移也可能使零样本检索/接地性能下降 >30%
└── 对应诊断：Module 2 — 跨模态遗忘检测 (Fig 7)
```

**解决方案分类**：

| 策略类别 | 核心方法 | 对抗的失败模式 |
|---|---|---|
| 多模态重放 | 显式/隐式记忆机制 | 跨模态特征漂移 |
| 跨模态正则化 | 约束模型更新保留对齐 | 模态对齐退化 |
| 参数高效适应 | LoRA / O-LoRA / 正交子空间 | 共享模块干扰 |

### 4.2 视觉幻觉的组件级分析

**Jing et al. (2025, arXiv:2505.01958)** 对 LLaVA 类 VLM 的每个组件进行幻觉来源分析：

```
幻觉来源定位：

Vision Encoder ─→ 不忠实的视觉表示 ─→ 输入端错误
     ↓
Projector ──────→ 跨模态对齐中的信息损失 ─→ 传输端错误
     ↓
LLM Backbone ──→ 过度依赖语言先验 ─→ 生成端错误
```

**对 Module 3 的直接启示**：
- **Modality Shortcut 检测**（遮蔽实验）→ 测试 LLM Backbone 的语言先验依赖（来源 3）
- **Description Collapse** → 测试 Projector 的信息压缩问题（来源 2）
- **Visual Hallucination** → 综合测试所有三个来源
- **建议新增**：Projection bypass 测试——直接比较 CLIP 嵌入与 LLM 隐状态，分离来源 2

### 4.3 视觉接地（Visual Grounding）

**arXiv:2509.10345** 建立了视觉接地在 VLM 中的核心地位：

- **接地定义**：不仅是回答正确，而是能精确定位和识别与自然语言描述对应的视觉区域
- **幻觉根源**：VLM 的幻觉源于对语言先验的过度依赖，随着生成更多 token，对视觉提示的依赖递减
- **接地不等于无幻觉**：后续研究表明接地目标与减少幻觉之间并不存在因果关系

**对 Module 3 Grounding Failure 检测的启示**：
- 不仅测试答案正确性，还要检测描述是否包含空间特异性细节 vs. 泛泛语言
- 遮蔽实验（真图/黑图/噪声图）正是验证视觉依赖性的标准方法

## 6. 总结与对项目的整体指导

### 6.1 文献为框架提供了什么

1. **理论合法性**：每个诊断模块的存在都有文献支撑，不是 ad-hoc 设计
2. **度量标准化**：采纳文献中已被验证的归一化遗忘率、对齐税量化公式
3. **解释词汇**：检测到遗忘时，可以用梯度干扰解释；检测到对齐税时，可以用参数子空间重叠解释；检测到幻觉时，可以归因到具体架构组件
4. **面试叙事**：从"我做了一个诊断工具"升级为"我将前沿遗忘和对齐研究的发现操作化为实用诊断管线"

### 6.2 文献揭示的关键研究缺口（本项目填补的）

| 缺口 | 本项目如何填补 |
|---|---|
| 大多数遗忘研究只关注单一阶段转换 | 我们覆盖 6 个阶段的完整 pipeline，含跨模态遗忘 |
| 对齐税研究很少涉及 VLM | 我们在 VLM 训练后也测量纯文本能力变化 |
| 幻觉研究多聚焦于大模型 | 我们在小模型上验证相同的诊断方法论是否适用 |
| 参数漂移分析缺乏与行为诊断的联合 | 我们将 M4 漂移与 M2 遗忘做因果推断 |
